# M2.3 Desktop, Mobile and Keyboard Review

## Desktop 1440×900

Gerçek Chromium viewport ve ekran görüntüsüyle ana/form/result yüzeyleri incelendi.

- Header ve navigation okunabilir.
- Form alanları, privacy uyarısı ve destek paneli görünür.
- GSS result kartı, gerekçe, next-step, source/guide alanı ve CTA'lar okunabilir.
- Desktop yatay taşma yok.
- Hukuk ve 404 route'ları ayrıca HTTP smoke ile geçti.

`M2_3_DESKTOP_1440X900=PASS`

## Mobile 390×844

Gerçek Chromium viewport sonucu:

```text
innerWidth=390
clientWidth=375
scrollWidth=387
HORIZONTAL_OVERFLOW=12px
```

Ekran görüntüsünde yatay scrollbar doğrulandı. Taşan yüzeyler GSS ana `card-panel` section ve sağdaki destek `aside/card-panel` bloklarıdır; sağ sınır yaklaşık 388px'e ulaşmaktadır.

Form, CTA, result ve navigation kullanılabilir olsa da zorunlu `horizontal overflow = 0` kapısı geçmedi.

```text
M2_3_MOBILE_390X844=FAIL
RELEASE_BLOCKING_RESPONSIVE_FINDING_COUNT=1
FINDING_ID=P1-RESPONSIVE-001
```

Bu bulgu runtime/security rollback koşulu değildir; prompt politikasına göre deployment aktif kalabilir fakat controlled beta kararı `NOT_READY` olmalıdır.

## Klavye ve focus

Gerçek Chromium üzerinde 28 öğelik Tab sırası kaydedildi:

- Header linkleri → gelir → hane → üç radio group → submit/reset → destek linkleri → footer.
- Tüm odaklarda `:focus-visible=true` ve solid outline görüldü.
- Shift+Tab, radio group'tan önceki hane alanına doğru döndü.
- Space checked radio davranışını korudu.
- Enter submit butonunu çalıştırdı ve `aria-live` NEEDS_INFO sonucunu üretti.
- Escape dialog/trap oluşturmadı; sayfada modal/dialog yok.
- Klavye trap ve mantıksız focus sıçraması görülmedi.

`M2_3_KEYBOARD_NAVIGATION=PASS`

## Erişilebilirlik kararı

Mobil taşma release blocker olarak kaydedildi ancak içerik kaybı, klavye trap veya kritik erişim engeli oluşturmadığı için kritik erişilebilirlik sınıfına alınmadı.

`M2_3_CRITICAL_ACCESSIBILITY_COUNT=0`
