# M2.2 Runtime Fix Design

## Hedef akış

```text
raw Request
→ body bytes
→ safe JSON parse
→ plain-object root guard
→ one canonical normalization function
→ required root/nested shape validation
→ backend proxy or deterministic local handler
→ stable public response
```

`normalizeEligibilityCheckRequest` tek normalizasyon kaynağıdır. Şunları zorunlu tutar:

- root değeri null, primitive veya array olamaz;
- `benefit_code` mevcut dört sözleşme kodundan biri olmalıdır;
- `facts` plain object olmalı, değerleri sözleşmedeki scalar/null sınırında kalmalıdır;
- opsiyonel `context` plain object olmalı ve bilinen alanları string olmalıdır;
- zorunlu alan uydurulmaz ve unsupported değer GSS'e varsayılanmaz.

Başarısız normalizasyon domain handler'a ulaşmaz. Route, payload veya exception ayrıntısı içermeyen kararlı yanıt üretir:

```json
{
  "message": "Uygunluk değerlendirme isteği geçerli değil.",
  "error": "invalid_request",
  "status": 400,
  "correlation_id": ""
}
```

HTTP 400, mevcut `ApiErrorResponse` ve istemci hata eşleme sözleşmesini korur. Bilinmeyen root alanları normalize çıktısına taşınmaz; structurally valid scalar fact uzantıları korunur. Geçerli GSS ve yaşlılık payload'ları aynı domain builder'lara gider.

## Değişmeyenler

- Route ve endpoint adı
- `EligibilityCheckRequest` / `EligibilityCheckResponse`
- UI, CTA, analytics ve assessment adapter'ları
- GSS ve yaşlılık karar mantığı/eşikleri
- Production, preview, domain, DNS ve secret durumu
