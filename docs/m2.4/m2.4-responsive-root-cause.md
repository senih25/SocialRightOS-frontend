# M2.4 Responsive Root Cause

## M2.3 kanıtı

M2.3 production incelemesinde 390×844 için `clientWidth=375`, `scrollWidth=387` ve 12px yatay taşma kaydedildi. Taşan yüzeyler GSS ana kartı ile destek `aside/card-panel` bloklarıydı.

## Kesin DOM/CSS nedeni

Mobilde pilot sayfaların dış grid'i açık bir sütun tanımlamıyordu. CSS Grid bu nedenle implicit `auto` track ve grid öğelerinde varsayılan `min-width:auto` kullandı. Sonuç, kaynak bağlantısı veya uzun kart içeriğinin min-content genişliği kullanılabilir alanı aştığında bütün track'i büyütebiliyordu.

```text
ROOT_CAUSE_COMPONENT=TR_GSS ve TR_OLD_AGE_PENSION dış sonuç/form grid'i
ROOT_CAUSE_FILE=src/app/gss-gelir-testi/GssToolPageClient.tsx; src/app/65-yas-ayligi-uygunluk-testi/OldAgeToolPageClient.tsx
ROOT_CAUSE_SELECTOR=main > div.grid; section.card-panel; aside.space-y-6
ROOT_CAUSE_PROPERTY=implicit grid track min-size auto + grid item min-width:auto
ROOT_CAUSE_MEASUREMENT=M2.3 390x844 clientWidth=375 scrollWidth=387 overflow=12px
AFFECTED_ROUTES=/gss-gelir-testi; /65-yas-ayligi-uygunluk-testi
```

Tam 320–1440 matrisi, aynı min-content kök nedeninin iki ek görünümünü ortaya çıkardı:

- 320px'de `.section-header` yatay flex düzeni: 21px taşma.
- İletişim kart grid'i ve uzun kanal değeri: 390px'de 85px, 320px'de 155px taşma.
- Gizlilik sayfasındaki uzun başlık/insan inceleme işareti: 320px'de 9px taşma.

`WHY_EXISTING_TESTS_MISSED_IT`: Önceki 195 test kaynak/iş mantığı sözleşmelerini doğruluyor, gerçek layout engine üzerinde `scrollWidth`, element bounds ve küçük viewport matrisi çalıştırmıyordu.
