# M2.1 — One-attempt Preview Execution Plan

## İzolasyon

1. Benzersiz isimli boş Vercel projesi oluştur.
2. Project ID ve team ID'yi CLI/API ile oku; yerel linkte birebir doğrula.
3. Project inspect, domains/aliases, env ve deployments API üzerinden sıfır custom domain, sıfır alias, sıfır env ve sıfır deployment doğrula.
4. Git integration bulunmadığını doğrula.

## Tek komut

```text
vercel deploy --yes --scope <verified-team>
```

Bu form resmî Vercel dokümantasyonundaki default Preview komutudur. `--prod` ve `--target` kullanılmaz.

## Anlık target gate

Smoke öncesi iki bağımsız kanıt:

1. `vercel inspect <deployment-id> --format=json`
2. `vercel api GET /v13/deployments/<deployment-id>`

İki kaynakta target `preview`, project/team doğru, state READY ve production/custom alias 0 olmalıdır. Aksi durumda smoke yapılmadan project removal rollback uygulanır.

## Rollback

- Tek deployment için `vercel remove <deployment-id>` veya izole projenin tamamını `vercel project remove <name>`.
- Sonrasında deployment list 0, alias/domain 0 ve URL 404/unavailable doğrulanır.
- İkinci deployment yapılmaz.

## Smoke

Target PASS sonrasında HTTPS, security headers, public/legal routes, 404, robots/sitemap, iki sentetik assessment, desktop 1440×900, mobile 390×844, analytics/privacy ve runtime marker doğrulanır.
