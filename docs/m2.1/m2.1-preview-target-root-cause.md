# M2.1 — Preview Target Root-cause Analysis

## Önceki olay

M2.0F sırasında `vercel deploy --yes --target preview` komutu kullanıldı. CLI sonuç nesnesi deployment'ı `target: production` olarak bildirdi ve proje alias'ı oluşturdu. Geçici proje tüm deployment'larıyla silindi; URL 404 ve active deployment 0 olarak doğrulandı.

## Resmî davranış

- Vercel, CLI'da `--prod` olmadan çalıştırılan düz `vercel`/`vercel deploy` komutunun Preview deployment oluşturduğunu belirtir: https://vercel.com/docs/deployments/environments
- Production deployment için açık `vercel deploy --prod` gerekir: https://vercel.com/docs/projects/deploy-from-cli
- Kurulu CLI 54.14.2 yardımında `--prod`, `--target=production` kısaltmasıdır; `--target` preview veya custom environment kabul edebilir: https://vercel.com/docs/cli/deploy
- Deployment bilgisi `vercel inspect <id|url> --format=json` ile okunabilir: https://vercel.com/docs/cli

## Hipotez matrisi

| ID | Hipotez | Durum | Kanıt | Risk / mitigation |
|---|---|---|---|---|
| H1 | Flag misuse/unsupported syntax | REJECTED | CLI yardım ve docs `--target preview` değerini destekliyor. | Yeni denemede daha basit default preview komutu kullanılacak. |
| H2 | Eski `.vercel` yanlış projeye bağlıydı | REJECTED | Önceki olayda proje açıkça yeni geçici projeye linklendi; sonrasında link silindi. | Yeni link ID/team birebir doğrulanacak. |
| H3 | Team/scope mismatch | REJECTED | Önceki çıktı ve güncel hesap aynı kişisel team scope'u gösteriyor. | Her komutta açık `--scope` kullanılacak. |
| H4 | Yeni projenin ilk deployment'ı production sınıflandı | CONFIRMED | M2.1 izole proje deployment öncesi 0 deployment idi; default Preview komutu sonrası inspect ve API `target=production` verdi. | Bu account/scope altında ilk CLI deploy güvenli Preview olarak kabul edilemez. |
| H5 | Target semantiği yanlış anlaşıldı | REJECTED | Resmî docs Preview/Production target ayrımını açık tanımlıyor. | `--target` yerine belgelenmiş default Preview kullanımı. |
| H6 | Production branch ayarı sınıflandırmayı etkiledi | REJECTED | M2.1 projesinde Git integration/production branch bağı yoktu; kaynak branch non-main idi. | Branch tek başına güvenlik kontrolü değildir. |
| H7 | Project creation production deployment oluşturdu | REJECTED | `project add` adımı deployment oluşturmadı; production kayıt deploy komutunda oluştu. | Create sonrası deployment count 0 doğrulaması. |
| H8 | CLI/API display inconsistency | REJECTED | Inspect JSON ve Deployments API aynı ID için production target, promoted state ve production environment gösterdi. | İki kaynaklı target doğrulaması zorunlu kalır. |
| H9 | Bilinmeyen platform davranışı | REJECTED | Davranış M2.1'de güvenli kontrollere rağmen yeniden üretildi ve H4'e indirildi. | Yeni CLI denemesi yapılmamalıdır. |

## Sınıflandırma

`ROOT_CAUSE_CLASSIFICATION=CONFIRMED_FIRST_DEPLOYMENT_PRODUCTION_CLASSIFICATION`

Bu Vercel account/scope altında boş projenin ilk CLI deployment'ı, resmî default Preview komutuna rağmen production sınıflandı. Güvenli ilerleme için başka CLI deploy denenmemeli; Vercel tarafında önceden doğrulanmış preview-only proje/ayar veya Vercel destek onayı gerekir.
