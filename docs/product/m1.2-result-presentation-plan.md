# M1.2 — Assessment Presentation Contract

Status: `PLAN_ACCEPTED_WITH_REVISIONS`

## Purpose

Add a passive presentation contract over M1.1 `AssessmentResult`. The mapper classifies display intent without calculating eligibility or changing backend status.

## First PR boundary

This PR contains only the presentation type, pure mapper, unit tests, and the result matrix. It does not integrate any tool, React component, route, CTA, threshold, payload, analytics event, or user-facing production copy.

## Contract

- Domain status remains `ELIGIBLE`, `NOT_ELIGIBLE`, or `NEEDS_INFO`.
- Presentation outcome is independently `POSITIVE`, `NEGATIVE`, `INCOMPLETE`, or `UNAVAILABLE`.
- Backend and legacy identity states remain distinct.
- Reasons, rule criteria, and missing information remain separate and ordered.
- Guidance becomes next-step input but is never promoted to an official channel.
- Official channels require explicit trusted input.
- Document checklist items require explicit copy-profile input.
- Validity exposes date, policy version, engine version, and a deterministic basis-version fallback.
- Benefit-specific data remains in the typed M1.1 extension.
- Raw backend responses and validation carriers are excluded.

## Copy boundary

The contract accepts a copy profile. This PR tests only synthetic profiles and does not move current GSS, old-age, birth-grant, or home-care copy. Real copy and UI integration require a separate pilot PR and explicit review.

## Failure behavior

Unsupported runtime status or mapper failure returns `UNAVAILABLE`. The fallback contains only the profile's safe copy and empty data sections; it never exposes an error, backend identifier, or invalid input value.

## Risks and mitigations

| Risk | Mitigation |
|---|---|
| Backend status confused with display outcome | Separate types and matrix tests |
| Guidance treated as official | Explicit trusted-channel argument only |
| Legacy home-care behavior generalized early | Keep typed extension; defer integration to M1.3 |
| Tool-specific fields inflate common model | Preserve benefit-specific extension |
| User-visible copy changes silently | Synthetic profiles only in this PR |
| Component scope expands | No React component or `src/app/**` changes |

## Pilot gate

A later PR may select one backend-backed tool for integration. It must preserve existing copy, routes, CTA, payload, analytics, accessibility, and behavior through explicit regression review. Four-tool migration is out of scope.

## Quality gates

Lint, all tests, production build/TypeScript, `git diff --check`, four-file allowlist, zero unexpected changes, clean worktree, draft PR, and green CI.

## Exclusions

Production, domain, Firecrawl, T03, backend changes, policy research, threshold changes, result storage, and UI integration.
