# M1.2 TR_OLD_AGE_PENSION pilot integration

## Scope

This pilot routes the existing 65 Yaş Aylığı result through the shared assessment and presentation contracts while preserving the current route, endpoint, payload, analytics, CTA and user-visible copy.

## Integration seam

`EligibilityCheckResponse → adaptEligibilityResponse → AssessmentResult<TR_OLD_AGE_PENSION> → buildAssessmentPresentationModel → old-age pilot adapter → existing inline result UI`

The API error path remains separate from presentation `UNAVAILABLE`. Raw backend responses and validation carriers do not enter the presentation model or UI.

## Fail-closed rule

When the presentation outcome is `UNAVAILABLE`, the result surface uses neutral approved copy and suppresses raw-status tone, label, badge, eligibility CTA and guidance. Accessible output must not contain a contradictory eligibility status.

## Preservation gates

- Route, endpoint and payload remain unchanged.
- CTA labels and destinations remain unchanged.
- Analytics events and payloads remain unchanged.
- Existing title, summary, disclaimer, status, error and loading copy remain unchanged.
- Guidance is never promoted to an official channel.
- Common contracts and GSS-specific files remain unchanged.

## Verification

The pilot requires adapter, copy, integration and accessibility regression tests plus lint, all tests, production build with TypeScript, diff check, allowlist verification and a clean worktree. Merge remains a separate human approval gate.

## Cloud decision

No Cloud Console, Cloud Shell, GCP, billing, production, staging, domain, Firecrawl or T03 action is required.
