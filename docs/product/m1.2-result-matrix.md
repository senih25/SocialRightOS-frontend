# M1.2 — Assessment Presentation Result Matrix

This matrix is the implementation contract for the passive mapper. Titles and summaries are supplied by synthetic profiles in the first PR.

| Input | Outcome | Identity | Evidence sections | Next steps / official channels | Fail-closed and accessibility expectation |
|---|---|---|---|---|---|
| `ELIGIBLE` | `POSITIVE` | Source-dependent | Keep reasons, rules, and missing information separate | Guidance is next-step input; official channels require trusted input | Status remains explicit; color is a later UI concern |
| `NOT_ELIGIBLE` | `NEGATIVE` | Source-dependent | Preserve order and severity without new decision logic | Same trust boundary | Never present as an official rejection |
| `NEEDS_INFO` | `INCOMPLETE` | Source-dependent | Missing information remains distinct from negative reasons | Same trust boundary | Must not appear positive or negative |
| Unsupported runtime status | `UNAVAILABLE` | `null` | Empty | Empty | Safe profile copy only; no technical detail |
| Mapper failure | `UNAVAILABLE` | `null` | Empty | Empty | Error and invalid payload are not exposed |
| Backend with decision ID | Status-derived | `VERIFIED_BACKEND` | Normal mapping | Normal mapping | ID state is not a new eligibility signal |
| Backend without decision ID | Status-derived | `MISSING_BACKEND_DECISION` | Normal mapping | Normal mapping | Do not convert to negative result |
| Legacy local result with null backend ID | Status-derived | `LOCAL_LEGACY` | Local reasons remain local | Explicit inputs only | Null backend ID is expected, not an error |
| Partial rule results | Status-derived | Source-dependent | Preserve provided rules in order | Unchanged | No missing rules are invented |
| Empty rule results | Status-derived | Source-dependent | Empty section signal | Unchanged | UI must omit empty section later |
| Guidance with no trusted channel | Status-derived | Source-dependent | Unchanged | Next steps populated; official channels empty | Guidance is not presumed official |
| Explicit trusted channel | Status-derived | Source-dependent | Unchanged | Official channel populated from argument | Link needs contextual label in UI integration |
| Explicit document checklist | Status-derived | Source-dependent | Unchanged | Checklist populated from profile | Must be labelled as preparation, not guarantee |
| Date-sensitive result | Status-derived | Source-dependent | Unchanged | Unchanged | `validAsOf` and `basisVersion` remain available |
| Birth-grant details | Status-derived | Source-dependent | Unchanged | Unchanged | Typed extension preserved without flattening |
| Legacy home-care threshold context | Status-derived | `LOCAL_LEGACY` | Unchanged | Unchanged | Threshold remains extension context, not new copy |

## Future component acceptance

- Outcome and status must be conveyed in text, not color alone.
- Reasons, rules, and missing-information sections require semantic headings and lists.
- Empty sections are not rendered.
- Result changes use an appropriate live region without moving focus unexpectedly.
- Links use contextual labels; mobile layout and keyboard access remain intact.
- User-specific assessment results are not indexable SEO content.
