# M2.0A — Mevcut Durum Envanteri

Tarih: 2026-07-16
Başlangıç main SHA: `35ee5ce890d40479625cac40a7b9a0a1c2316632`

## Teknik temel

- Next.js 16.2.10 App Router, React 19.2.4, TypeScript 5.9.3 ve Tailwind CSS 4.
- Node 22 ve npm 11 sabitlenmiş; `package-lock.json` mevcut.
- Standalone Next.js çıktısı, çok aşamalı Dockerfile ve 8080 portlu başlangıç betikleri mevcut.
- GitHub Actions; `npm ci`, TypeScript, lint ve tüm testleri çalıştırıyor. CI henüz production build, diff check veya secret scan çalıştırmıyor.
- 37 test dosyasında 159 Node test-runner testi mevcut. Ayrı browser/component/a11y test framework'ü yok.

## Ürün yüzeyleri

- Ana ürün: ana sayfa, `/start`, blog, yöntem, hakkımızda, iletişim, yasal uyarı ve marka/kurucu sayfaları.
- Araçlar: doğum yardımı, evde bakım, gelir testi, GSS ve 65 yaş aylığı.
- M1.2 pilotları: `TR_GSS` ve `TR_OLD_AGE_PENSION` ortak `AssessmentPresentationModel` sözleşmesine bağlanmış durumda.
- Admin yüzeyleri `/admin/**` altında; robots ile index dışı. Admin studio taslakları localStorage kullanıyor.
- API geçidi `/api/[...path]`; backend taban URL'si sunucu değişkenlerinden seçiliyor. İstemci `checkEligibility` üzerinden typed response tüketiyor.

## Değerlendirme ve sunum sözleşmeleri

- `EligibilityCheckResponse` → `adaptEligibilityResponse` → `AssessmentResult` → `buildAssessmentPresentationModel` akışı mevcut.
- Backend status ile sunum outcome'u ayrılmış; desteklenmeyen/malformed durumlar `UNAVAILABLE` fail-closed sonucuna gider.
- Ham backend response ve validation carrier presentation modeline taşınmıyor.
- GSS ve 65 yaş pilot adaptörleri kullanıcı metnini, CTA'yı, endpoint'i, payload'ı ve analytics sözleşmesini koruyan regresyon testlerine sahip.
- API error yolu ile presentation `UNAVAILABLE` yolu ayrı tutuluyor.

## UX ve erişilebilirlik

- Form gönderimi sırasında buton disable ediliyor; loading metni gösteriliyor.
- Sonuç bölgelerinde `aria-live="polite"`, semantik başlıklar ve listeler var.
- Tri-state alanlar fieldset/legend/radio kullanıyor.
- Global `error.tsx`, `loading.tsx` ve `not-found.tsx` yok.
- API/validation hata panelinde `aria-live`/alert semantiği ve input-error association yok; backend alan adları kullanıcıya gösterilebiliyor.
- Mobile-first sınıflar yaygın, ancak otomatik responsive/keyboard/screen-reader smoke testi yok.
- `AccessibleButton` görünür focus ring'i bastırıyor; kritik erişilebilirlik incelemesi gerekli.

## SEO/GEO

- Root metadata, metadata base, canonical, Open Graph, Twitter card, robots ve sitemap mevcut.
- Preview/non-production ortamları index dışı ve sitemap boş.
- JSON-LD yardımcıları var; kontrollü `JSON.stringify` çıktısı kullanılıyor.
- Bazı route'lar sitemap dışında; route bazında metadata/heading bütünlüğü otomatik doğrulanmıyor.

## Güvenlik ve gizlilik

- CSP, HSTS, nosniff, DENY frame, referrer, permissions, COOP ve CORP başlıkları mevcut.
- CSP script/style için `unsafe-inline` içeriyor; nonce/hash stratejisi yok.
- İstemci secret'ı tespit edilmedi; örnek env yalnız public site/API URL'leri içeriyor.
- Analytics browser event/dataLayer adaptörü kullanıyor. Sağlık, kimlik veya tam form payload'ı gönderen kod gözlenmedi.
- `income-evaluation-analytics.ts` browser console'a olay yazıyor; production logging politikası net değil.
- Admin studio localStorage kullanıyor; hassas veri yasağı teknik guard ile uygulanmıyor.
- Otomatik secret scan ve client-bundle secret doğrulaması yok.

## Legal, operasyon ve observability

- Yasal uyarı, yöntem ve iletişim sayfaları var.
- Ayrı KVKK/gizlilik, kullanım koşulları ve kaynak-güncellik politikası sayfaları yok.
- Hukuki metinler için `human legal review required` durumu henüz ürün içinde kayıtlı değil.
- Uygulama error boundary, sanitized error adapter, release SHA/environment marker ve gerçek monitoring provider içermiyor.
- Staging/rollback belgeleri mevcut; doğrulanmış güncel preview smoke kanıtı yok.
