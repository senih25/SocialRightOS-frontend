## TR_GSS uçtan uca akış

```text
GssToolPageClient
→ buildGssPayload(form, crypto.randomUUID())
→ checkEligibility
→ POST /api/v1/eligibility/check
→ EligibilityCheckResponse
→ adaptEligibilityResponse<TR_GSS>
→ buildAssessmentPresentationModel
→ buildGssPilotPresentationViewModel
→ mevcut inline result UI
```

Korunan sözleşmeler: route, endpoint, payload alanları, CTA metni/hedefi, `tool_opened`, `form_started`, `form_submitted`, `result_received` analytics olayları. API error ayrı hata paneline; presentation mapper hatası ise güvenli `UNAVAILABLE` sonucuna gider.

## TR_OLD_AGE_PENSION uçtan uca akış

```text
OldAgeToolPageClient
→ buildOldAgePayload(form, crypto.randomUUID())
→ checkEligibility
→ POST /api/v1/eligibility/check
→ EligibilityCheckResponse
→ adaptEligibilityResponse<TR_OLD_AGE_PENSION>
→ buildAssessmentPresentationModel
→ buildOldAgePilotPresentationViewModel
→ mevcut inline result UI
```

GSS ile aynı error/presentation ayrımını ve fail-closed davranışı kullanır. Benefit extension ortak alanlara açılmaz.

## Durum matrisi

| Durum | Presentation outcome | UI davranışı |
|---|---|---|
| `ELIGIBLE` | `POSITIVE` | Olumlu ön değerlendirme; kesin karar değil |
| `NOT_ELIGIBLE` | `NEGATIVE` | Olumsuz yön; yeniden kontrol/rehberlik |
| `NEEDS_INFO` | `INCOMPLETE` | Eksik bilgi ve sonraki adımlar |
| Unsupported/mapper failure | `UNAVAILABLE` | Raw status/badge/CTA/guidance bastırılır |
| API/network failure | Presentation dışı | Güvenli istek hata paneli |

## Kritik eksik durum yüzeyleri

- Global unexpected-error boundary yok.
- Route-level loading ve not-found deneyimi yok.
- Validation errors inputlarla programatik bağlı değil.
- Error panel screen-reader canlı duyurusu tanımlı değil.
- Responsive ve keyboard davranışı yalnız sınıf/kod incelemesiyle korunuyor; otomatik smoke yok.
